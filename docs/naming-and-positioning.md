## Chosen name: Fermata

Fermata names the runtime boundary: hold before acting. The project centers on the
moment where an agent proposal is not yet an external-world effect. The runtime
must admit, verify, approve, and only then commit.

Core chorus:

> Agents may propose; only governed effects may commit.

## Adjacent names retained as internal concepts

- `continuo` — the substrate metaphor: governed runtime beneath orchestrators.
- `proviso` — the policy/condition metaphor: action is allowed only under explicit conditions.
- `katas` — the practice/examples track, not the repo name.
- `praxis` — the practice-through-action ethos, useful for docs/examples but too broad as a repo identity.
